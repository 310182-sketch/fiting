## Plan: 本地健身紀錄 PWA 實作藍圖

我們會把目前幾乎是空白的 `fiting` 專案，逐步變成一個以 React + TypeScript + Vite 為基礎、支援安裝與離線的健身紀錄 PWA。重點是：所有訓練數據存在本機（IndexedDB/Dexie），無需後端；MVP 先完成「開始訓練 → 記錄組數 → 查看歷史」的完整閉環，再考慮視覺優化與進階統計。

### Steps
1. 初始化前端專案骨架：在 `fiting` 建立 Vite React+TS 專案，整理 `src/` 結構與基本入口組件。
2. 加入 Tailwind CSS：設定 `tailwind.config`、全域樣式檔，建立簡單的行動版 Layout（頂部標題＋底部導覽列）。
3. 設定 PWA 能力：安裝 `vite-plugin-pwa`，建立 `manifest`（app 名稱、icon、顏色），啟用基礎 Service Worker 快取。
4. 設計與實作本地資料庫層：用 Dexie 定義 `exercises`、`workouts`、`sets` 三個資料表與型別，並建立 API 包裝（CRUD 函式）。
5. 規劃與建立路由頁面：用 React Router 建立「Dashboard（首頁）」、「開始訓練」、「歷史紀錄」、「設定」等頁面骨架與基本導航。
6. 完成訓練流程 MVP：在「開始訓練」頁實作選擇動作、輸入重量/次數/組數、儲存到 IndexedDB，並在「歷史紀錄」頁讀取與顯示細節。
7. 實作本地備份功能：在「設定」頁加入資料匯出（JSON 下載）與匯入（JSON 上傳解析寫回 DB）的流程。
8. 優化體驗與細節：加入簡單統計（例如本週總訓練次數）、錯誤處理（DB 失敗提示）、以及行動裝置安裝教學提示。

### Further Considerations
1. 預設動作：建議預先寫入一組常見動作清單（如深蹲、臥推、硬舉、肩推等），是否要以「力量訓練」為主還是也包含有氧（跑步、騎車）？
2. 訓練流程形式：你偏好「先選課表再記錄」還是「直接開始訓練、邊做邊加動作」的操作流程？這會影響畫面設計。
3. 統計與視覺化：MVP 想要到什麼程度？例如僅列表＋簡單數字，還是要有圖表（可考慮後續用 Recharts 等套件）。
